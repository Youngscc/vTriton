## Learnings
- Real chunk_kda profiles as Task Type MIX_AIC (mixed cube), NOT AI_CORE.
  parse_kernel_time_us dropped these (matched only AI_CORE/AICORE) while
  parse_component_durations kept them — a silent divergence only real data
  exposed. Fixed via shared _is_aicore_task() (AI_CORE/AICORE/MIX_AIC/MIX_AIV/
  AI_VECTOR_CORE/AIV).
- Remote is aarch64; the x86-64 tritonsim-hivm ELF cannot run there, so an
  offline chunk_kda des.json still needs an NPU x86 host (or a native build).
- read_msprof_csv now summarises skipped N/A rows once (real CSVs have ~84).
- vtriton-verify env: run pytest with TORCH_DEVICE_BACKEND_AUTOLOAD=0
  (torch_npu auto-load crashes without a device); coarse WSL mtime can serve a
  stale .pyc to inspect.getsource — clear __pycache__ if a source-inspect test
  flakes.
